CodeStyle

What is Code Styling?

This site is dedicated to showcasing some common and useful styling conventions for clean, readable code. Following these conventions ensures that others can read your code without having call you to translate for them.

Why Style Your Code?

Styling is especially useful in situations where you are contributing to projects with multiple members, and want to keep a consistent layout throughout your codebase. This prevents your projects from looking like they were frankenstein'd together by multiple contributors, which makes it easier for future contributors or even you at a later date.

For Example:

<html><body><div><h1>My Header</h1><p>My paragraph text, really important</p></body></html>

This may look fine to an HTML interpreter but to a human, it's obviously extremely difficult to read. The reason most websites' source code doesn't look like this (unless they're trying to obfuscate data) is because their developers followed common practices for making their code look readable.

This is a bit more readable:

<html>
 <body>
   <h1>My Headers</h1>
   <p>
    My paragraph text, really important
   </p>
 </body>
</html>
