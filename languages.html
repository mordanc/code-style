Languages

Intro

In this section I'll talk about styling/naming conventions for specific languages. These practices are either created by an authority in the language, or broadly used by the community.

Python

Probably the most well-documented guide for Python style is PEP8 and is maintained by the Python Software Foundation (PSF).

PEP8 was assembled from Python author Guido van Rossum's original style guide, with additions from other contributors. It represents probably the most comprehensive guide for core Python libraries, and is a good starting point for anyone trying to make their Python script readable.

With a language as accessible and easy to use as python, it can be tempting to just spit out code line-by-line without much thought, but after a while your code will get much longer and you'll thank yourself for organizing it.

One of Guido's key insights is that code is read much more often than it is written.

- PSF
