Common Coding Conventions

While every programmer has their own preferences when it comes to how their code should look, there are some conventions that are universally recognizable.

Constants

Constants are variables whose values will not change over the course of a program's lifecycle, and should generally be declared in a file separate from the main functionality of your program. Most programmers will be able to recognize a variable as a constant if it is fully capitalized, for example: MY_CONSTANT.
